# Markup Edit — 인라인 마킹 수정

사용자가 파일에 직접 남긴 `<! !>` 마킹과 `--` 지시를 파악하고, 지시에 따라 수정한 뒤, 마킹 태그를 깨끗이 제거하는 워크플로우.

## 마킹 문법

```
<! 수정할 텍스트 !> --수정 지시
```

| 요소 | 문법 | 설명 |
|------|------|------|
| 수정 대상 | `<! ... !>` | 수정이 필요한 문장/문단. 여러 줄 가능 |
| 수정 지시 | `-- 내용` | 마킹 직후에 붙는 수정 방향 |

마킹은 한 줄일 수도, 여러 줄에 걸칠 수도 있다:
```
단일: <! 이 문장이 어색하다. !> --자연스럽게 수정

복수: <!
"대사 1"
서술
"대사 2"
!> --대사 톤 수정
```

## 실행 절차

### Step 1: 파일 읽기 + 마킹 스캔

대상 파일을 Read로 전체 읽는다. `<! ... !>`와 `--` 패턴을 모두 찾아낸다.

### Step 2: 마킹 목록 제시 (필수)

수정 전에 반드시 사용자에게 요약 테이블을 보여준다. 사용자가 확인해야 작업을 시작할 수 있다.

```
| # | 위치 | 대상 텍스트 (요약) | 지시 |
|---|------|-------------------|------|
| 1 | L행번호 | 마킹된 내용 앞부분... | -- 뒤의 지시 내용 |
| 2 | L행번호-행번호 | 여러 줄이면 범위 표시 | 지시 내용 |
```

사용자가 "작업 시작", "ㅇㅇ", "맞아" 등으로 확인하면 진행.

### Step 3: 수정 실행

각 마킹을 Edit 도구로 수정한다.

**핵심: 수정과 태그 제거를 한 번에 처리한다.**

- `old_string`에 `<! !>` 태그와 `--` 주석을 포함
- `new_string`에 수정된 본문만 넣음 (태그 없이)
- 이렇게 하면 수정과 태그 제거가 동시에 이루어져 태그가 남을 위험이 없다

**연쇄 수정**: 하나의 수정이 파일 내 다른 부분에 영향을 줄 경우, 해당 부분도 함께 수정한다.
- 금액 변경 → 합계, 계산식, 비교 문장 등 전부 업데이트
- 캐릭터 설정 변경 → 하단에서 해당 캐릭터를 참조하는 모든 곳 업데이트
- 사실관계 변경 → 같은 사실을 전제로 한 서술 전부 업데이트

연쇄 수정은 마킹되지 않았더라도 논리적 일관성을 위해 반드시 수행한다.

### Step 4: 태그 잔여 확인 (필수)

모든 수정 완료 후 Grep으로 `<!` 패턴을 검색한다.

```
Grep: pattern="<!", path=대상파일
```

- 결과 없음 → 정상 완료
- 결과 있음 → 남은 태그 즉시 제거

이 단계를 건너뛰면 안 된다. 태그가 원고에 남으면 사고다.

### Step 5: 결과 보고

수정 결과를 요약 테이블로 보고한다:

```
| # | 수정 내용 |
|---|----------|
| 1 | 무엇을 어떻게 바꿨는지 한 줄 요약 |
| 2 | 연쇄 수정이 있었으면 함께 언급 |
```

## 원칙

1. **태그 잔류 절대 금지**: 수정 후 `<! !>`, `--` 주석이 파일에 남으면 안 된다
2. **연쇄 수정 필수**: 수정이 다른 곳에 영향을 미치면 마킹되지 않았어도 함께 수정
3. **확인 후 실행**: Step 2의 테이블을 보여주고 사용자 확인 후 작업 시작
4. **원문 존중**: 마킹되지 않은 부분은 건드리지 않음 (연쇄 수정만 예외)
5. **지시 해석**: `--` 뒤의 지시가 짧더라도 맥락을 파악하여 적절히 수정. "비문임"이면 문법 수정, "삭제"면 해당 부분 제거, "톤 수정"이면 문체 변경 등
