# Writing Process Rules - 집필 워크플로우 규칙

현대 타워 판타지 먹치킨물 소설 작성의 전/중/후 프로세스와 파일 업데이트 트리거를 정의한다.

## Before Writing (작성 전 - 필수)

1. `00-핵심관리/현재작업상태.md` 읽기
2. 관련 캐릭터 파일 확인 (`02-캐릭터/`)
3. 현재 화 컨텍스트 파악
4. `03-플롯/플롯홀.md`, `03-플롯/복선추적.md`, `03-플롯/인물관계도.md` 검토
5. **능력 현황 확인** — 생체모방 획득 목록, 합성 현황, 최근 사용 능력 점검
6. **타워 층수 확인** — 현재 진행 중인 타워 층수, 환경, 몬스터 유형
7. **파티원 상태 확인** — 소라/해민/기택 현재 감정, 부상, 장비 상태
8. `03-플롯/서사줄기추적.md` 최근 5화 균형 확인 → 이번 씬 줄기 결정
9. 이전 2-3 씬 원고 읽기 (문맥 연속성 및 톤 확보)
10. **능력 업그레이드 빈도 점검** (`ability-upgrade-current.md`) — 직전 소(小) 업그레이드 이후 화수 확인. 4화 초과 시 이번 화에 소 업그레이드 필수 배치
11. **화 성격 유형 선언** — 포석형/돌파형/여운형/인간형/전환형 중 하나를 명시적으로 선택. 같은 유형 3회 연속 금지. 인간형: 5화당 1회 이상 필수. 전환형: 소아크당 2회 이상 필수.
12. **씬 클라이맥스 위치 선언** — 전면배치/중간배치/후면배치 중 하나를 명시적으로 선택. 후면배치만 연속 4회 금지. 5화당 전면배치 1회, 중간배치 1회 이상 필수.
13. **쾌감 패턴 연속 확인** (`entertainment-density.md` P3 참조) — 직전 화와 동일한 E패턴 사용 금지. E1/E5/E6/E7 저사용 여부 점검.
14. **엔딩 방식 확인** (`entertainment-density.md` P4 참조) — 직전 화 엔딩 방식 확인. 동일 방식 금지. 끊김형/대화잔향형 의무 배분 현황 점검.
15. **전투씬 포함 여부 확인** → `style-guide-battle.md` Read 준비

## 화 분량 기준 (Episode Length Standard) — 필수 준수

### 분량 목표 (한글 글자 수 기준)
| 단위 | 목표 | 최솟값 | 비고 |
|------|------|--------|------|
| 화 전체 | 6,000~7,000자 | **5,000자** | 미달 시 즉시 씬 확장 |
| 씬 1개 | 2,000~3,500자 | 1,500자 | 2-3씬 구성 기본 |

### 예외 케이스
- **클라이맥스 화** (소아크 마지막, 보스전): 8,000자 이상 허용
- **전환/연결 화**: 최소 4,500자 (이하 금지)

### 분량 확장 방법 (단순 패딩 금지 — 밀도 있는 확장만)
1. **내면 묘사 확장**: 태봉의 생물학 지식·해충방제 비유·판단 근거를 풀어쓰기
2. **능력 해설 상세화**: 생체모방 메커니즘 → 적용 과정을 독자가 이해하도록 서술
3. **대화 비트 확장**: 대화 전후 행동·신체 반응·코미디 비트 추가
4. **씬 전환 여운**: 현재 씬의 결과·감정·분위기를 정리하는 단락 추가
5. **감각 묘사 충실화**: 시각 외 청각·촉각·후각·미각을 충분한 문장으로 서술

### 분량 점검 (퇴고 체크리스트 추가 항목)
- [ ] 화 총 글자 수 확인 → 5,000자 미만 시 확장 필수
- [ ] 씬별 글자 수 확인 → 1,500자 미만 씬 존재 시 확장

## During Writing (작성 중 - 준수)

1. 씬 내 POV 일관성 유지 (전환 금지)
2. 캐릭터별 말투 적용 (캐릭터 파일의 대화 스타일)
3. 오감 활용 감각적 묘사 (최소 3개 감각)
4. 대화, 행동, 내면 묘사 균형
5. **코미디 비트 1개 이상** — 씬당 최소 1개 유머 요소 배치
6. **페널티 묘사** — 생체모방 사용 후 반동(배고픔/외형변화/졸림) 묘사 누락 금지
7. 마크다운 형식 사용 금지 (원고 내)
8. 자연스러운 산문 형식 유지
9. **대화 노이즈 최소 3회** — 끊김/딴소리/비완성 문장/오해/침묵
10. **능력 일관성** — 이미 사용한 능력의 효과/한계가 이전 화와 일치하는지 확인
11. **능력 업그레이드 에피소드 배치** — `ability-upgrade-current.md` 참조, 해당 화에 적합한 소 업그레이드 씬 내 통합
12. **전투씬 원칙 준수** — 능력 쇼케이스 6단계, 합성 시퀀스, 파티 전투 역할 분담. `style-guide-battle.md` 참조

## After Writing (작성 후 - 필수)

1. `00-핵심관리/진행상황.md` 업데이트
2. 감정 추적 기록 (`05-심리추적/감정추적/`)
3. 새로운 복선, 캐릭터 발전, 세계관 세부사항 기록
4. 플롯 일관성, 캐릭터 목소리, 페이싱 최종 확인
5. `03-플롯/서사줄기추적.md`에 이번 씬/화 줄기 비율 기록
6. **퇴고 실행 (필수, 예외 없음)** — 초고 완성 즉시 1차수정(퇴고)을 진행한다. 별도 지시 없이도 자동으로 수행.
7. **인간화 리뷰 실행 (필수)** — 점수 70 미만 시 해당 부분 즉시 수정 후 재검사.
8. **능력 업그레이드 기록** — `ability-upgrade-current.md` 기록 형식에 따라 트랙/규모/레퍼토리 코드/4요소 체크 기록.

### 퇴고 체크리스트 (1차수정 필수 항목)
- [ ] em-dash(—) 화당 5회 이하 확인, 4조건 미충족 시 대체
- [ ] 메타 서술 라벨 제거 ("능력 해석이 시작됐다" 등 선언형)
- [ ] 독자 설명 라벨 제거 (독자를 위해 캐릭터 행동을 설명하는 문장)
- [ ] 중복 설명 제거 (앞 문장이 이미 전달한 내용을 다음 문장에서 반복)
- [ ] 에피소드 종결 패턴 — 직전 화와 동일한 방식 금지
- [ ] 인간화 규칙 위반 패턴 확인 (수치 시간, 동일 신체 표현 반복 등)
- [ ] POV 일관성 씬 내 유지 여부
- [ ] 대화 노이즈 최소 3회 확인
- [ ] 코미디 비트 존재 확인 — 씬당 최소 1개
- [ ] 능력 페널티 묘사 확인 — 생체모방 사용 후 반동 누락 없는가
- [ ] 능력 업그레이드 에피소드 점검 — 직전 4화 내 소 업그레이드 0회면 이번 화에 추가 필수
- [ ] **문장 단편화 점검** — SF01-SF05 위반 없는가 (`style-guide.md` 참조)

## File Update Triggers

| Event | Update Target |
|-------|--------------|
| 새 화 완성 | `00-핵심관리/진행상황.md` |
| 캐릭터 발전 | 캐릭터 파일 + 성장 궤적 |
| 세계관 추가 | 관련 세계관 파일 |
| 플롯 포인트 해결 | `03-플롯/복선추적.md` |
| 새 복선 설치 | `03-플롯/복선추적.md` |
| 감정 변화 | `05-심리추적/감정추적/[캐릭터].md` |
| 쾌감 패턴 사용 | 씬 메타데이터에 패턴 ID 기록 |
| 세력 변동 | `03-플롯/인물관계도.md` |
| 전투 결과 | 타워 진행 현황 업데이트 |
| 능력 성장 | 캐릭터 파일 능력 섹션 + `ability-upgrade-current.md` |
| 합성 발견 | 생체모방 합성 목록 업데이트 |

## Daily Routine

### 시작 (10분)
1. 대시보드 확인
2. 현재작업상태 읽기
3. 관련 파일 열기
4. 목표 설정

### 집필 시간
1. 씬 체크리스트 확인
2. 목표 글자수 작성
3. 일관성 유지
4. 감정선 추적

### 마무리 (15분)
1. 진행상황 업데이트
2. 파일 백업 (Git commit)
3. 내일 계획

## Version Naming

```
ep-[NNN]_v[N]_[상태].md

상태: 초고 | 1차수정 | 2차수정 | 베타리딩 | 최종
예시: ep-001_v1_초고.md → ep-001_v2_1차수정.md
```
